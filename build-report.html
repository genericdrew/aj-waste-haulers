Section 6

Stage 7 — What Review Caught

Three review passes ran: a code/copy/character audit of the built layouts, and two rounds of visual review on 36 rendered screenshots. Visual review round 1 caught a CSS-import blocker that static analysis missed — because the code read fine but the browser couldn't see the tokens. Round 2 (post-fix) found only polish items.

BLOCKER · FIXED

CSS @import "_tokens.css" not resolving through Hugo's fingerprint pipeline

Caught by visual review round 1 · Fix applied · Round 2 verified resolved

The stylesheet used a plain-CSS @import to pull in the token file. Hugo's resources.Fingerprint copies the file byte-for-byte, so the import statement survived into the minified output — where it pointed to a path that didn't exist on the served site. Result: zero palette tokens reached the browser, the entire site rendered as unstyled black-on-white.

Fix Load the two CSS files as separate Hugo resources, then bundle with resources.Concat before fingerprinting. The tokens now inline into the single served site.css.
BLOCKER · FIXED

tel: / sms: / mailto: URLs rendering as #ZgotmplZ

Caught by code review · Fix: pipe protocol URLs through | safeURL

Hugo's template security silences any URL starting with a non-HTTP scheme unless it's explicitly marked safe. Every phone and SMS link across the site was dead on the first build. Applied | safeURL at 12 call sites across header, footer, mobile CTA bar, hero, CTA banner, service detail, and contact templates.

ERRORS · FIXED

Hardcoded display strings in 4 layouts (G07 violation)

Caught by code review · Moved to data/content.yaml → ui

Four strings were baked into templates: "Services" eyebrow, "Frequently asked", "Call or request a quote", "Top Service Areas", and the city-specific "Services we bring to X". Moved every one into content.yaml under a new ui: key.

ERRORS · FIXED

Interior-page SEO misses (slug titles, wrong schema)

City pages + services + About page

City pages rendered <title>dripping-springs | AJ</title> — the raw slug. All 10 city + service detail pages got explicit meta_title + meta_description front matter. About-page schema was restructured from Organization to LocalBusiness + Person × 2 per CLAUDE.md §17.

WARNINGS · DOCUMENTED

Polish items deferred to v1.1

Character Bible IS-NOT nits, palette edge cases

Round 2 visual review flagged: trust-strip legibility at 768px could improve, "Schedule a Metered Pumping" CTA wraps on 375px mobile, founder-card terracotta top-border drops at mobile-375, city-card shadows missing on service-area hub. None blocking; all logged in CHANGELOG.md for a polish pass.

The bottom line

A 23-page, research-driven, schema-instrumented Hugo site was delivered in ~2 hours of wall-clock for $32.25.

Model routing held: 70% Opus, 30% subagent. Parallel batching cut subagent wall-clock nearly in half. Prompt caching saved ~92% on subagent token bills. Visual review caught a CSS pipeline bug that static code review alone missed — demonstrating that review agents need to see the rendered output, not just the source. The framework shipped a production-grade site with only content assets (photography, license numbers, Google reviews) and a form endpoint pending as pre-launch TODOs.
